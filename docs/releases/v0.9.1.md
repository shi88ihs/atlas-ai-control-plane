# Atlas v0.9.1 – Infrastructure Stabilization Release

## Overview
Atlas v0.9.1 focuses on hardening the deployment environment, establishing a professional open-source repository structure, and preparing for the final production web cutover. This release transitions the project from an internal prototype into a stable, GitOps-driven operations platform.

## Highlights
- **Mission Control Dashboard:** Laid the foundation for single-pane observability across the fleet.
- **Documentation Restructure:** Overhauled the repository to match portfolio-grade open-source standards, introducing comprehensive architecture, deployment, and security guides.
- **Memory Optimization Profile:** Identified and isolated memory-heavy processes, recovering over 400MB of active RAM.
- **Backup and Release Workflow:** Implemented automated snapshotting, checksum validation, and release artifact generation.

## Operational Improvements
- Established a rigorous `.gitignore` policy and a security publication checklist to prevent secret leakage.
- Finalized the migration from legacy Docker orchestration to native `systemd` user services, establishing `hermes-gateway.service` as the canonical source of truth.
- Conducted a full network and runtime capability audit, proving Atlas runs securely without root privilege escalation.

## Known Work Remaining
- **DNS Cutover:** Namecheap DNS records must be repointed to the AWS Elastic IP.
- **Production Web Stack:** Caddy reverse proxy and Vaultwarden containers are configured but awaiting DNS propagation to acquire Let's Encrypt certificates.
- **Vultr Retirement:** The legacy Vultr server remains online pending final verification of the AWS cutover.

## Looking Ahead
As we approach v1.0.0 (Public Preview), the focus will shift to finalizing the Mission Control dashboard, expanding multi-node fleet management capabilities, and completing the migration of all public-facing services (Vaultwarden, personal site) exclusively to the Atlas AWS infrastructure.
